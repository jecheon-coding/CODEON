학생 대시보드의 전체 구조는 유지하되, '오늘의 숙제'는 현재의 '오늘 학습 현황' 카드 안에 억지로 넣지 말고 별도의 실행형 리스트 카드로 분리해줘. 핵심 목표는 학생이 대시보드에 들어오자마자 '오늘 몇 개 남았는지'와 '지금 어떤 문제를 바로 풀어야 하는지'를 직관적으로 이해할 수 있게 만드는 것이다.

[전체 방향]
- 현재 상단 배너, 요약 카드, 마지막 학습, 오늘 학습 현황 구조는 유지
- 다만 '오늘의 숙제'는 요약 정보와 실행 목록의 성격이 다르므로 별도 카드로 분리
- 대시보드는 숫자 요약과 실제 행동 유도를 구분해서 보여줘야 함
- 정보는 많아 보여도 학생이 바로 이해할 수 있게 위계를 분명히 정리

[핵심 원칙]
- '오늘 학습 현황'은 요약 카드로 유지
- '오늘의 숙제'는 실제로 눌러서 문제를 푸는 실행 리스트 카드로 분리
- 한 카드 안에 진행 요약과 숙제 목록을 동시에 길게 넣지 말 것
- 학생이 '지금 무엇을 해야 하는지'가 가장 먼저 보이게 만들 것

--------------------------------------------------
1. 오늘의 숙제 카드 위치
--------------------------------------------------

현재 화면 기준으로 오른쪽 영역을 두 개 카드로 나눠줘.

권장 구조:
- 오른쪽 상단: 오늘 학습 현황
- 오른쪽 하단: 오늘의 숙제

즉, 기존의 오른쪽 하단 영역을 '오늘 학습 현황' 하나로 크게 쓰지 말고,
상단에는 요약형 카드,
하단에는 숙제 리스트형 카드로 분리해줘.

왼쪽 하단의 '마지막 학습' 카드는 그대로 유지해도 된다.

--------------------------------------------------
2. 오늘 학습 현황 카드 역할 재정의
--------------------------------------------------

'오늘 학습 현황' 카드는 계속 유지하되, 이 카드는 어디까지나 요약 정보만 보여주는 카드로 정리해줘.

포함할 정보 예시:
- 오늘 숙제 진행률 (예: 1/3)
- 총 제출 수
- 정답 수
- 오답 수
- 정답률
- 간단한 동기부여 문구

이 카드 안에는 긴 숙제 목록을 넣지 말아줘.
'추천 문제' 정도의 보조 요소는 가능하지만, 숙제 리스트 자체는 넣지 않는다.

즉, 이 카드는 '현황 요약'만 담당하게 해줘.

--------------------------------------------------
3. 오늘의 숙제 카드 추가
--------------------------------------------------

오늘 학습 현황 카드 아래에 별도의 '오늘의 숙제' 카드 또는 '오늘 해야 할 문제' 카드를 새로 만들어줘.

카드 제목 예시:
- 오늘의 숙제
또는
- 오늘 해야 할 문제

우측 상단:
- 전체보기 버튼 또는 더보기 링크

카드 설명:
- 오늘 제출해야 하는 문제를 확인하고 바로 이어서 풀 수 있도록 구성

--------------------------------------------------
4. 오늘의 숙제 리스트 구성
--------------------------------------------------

리스트는 너무 길지 않게 구성하고, 기본적으로 3개 정도만 노출해줘.
숙제가 더 많으면 하단에 '전체 숙제 보기' 또는 '남은 숙제 2개 더 보기' 같은 방식으로 연결해줘.

각 행에는 아래 항목을 보여줘.
- 문제 제목
- 소속 과제명 또는 주제명
- 상태 배지
- 우측 액션 버튼

상태 배지 예시:
- 미제출
- 진행중
- 완료

액션 버튼 예시:
- 바로 풀기
- 이어서 하기
- 다시 보기

우선순위 정렬은 아래 기준으로 해줘.
1. 미제출
2. 진행중
3. 완료

즉, 학생이 지금 해야 할 숙제가 가장 먼저 보이게 해줘.

--------------------------------------------------
5. 숙제 카드의 UI 방식
--------------------------------------------------

한 행 한 행이 너무 큰 카드처럼 보이지 않게, 리스트형으로 간결하게 구성해줘.
문제 제목이 가장 강조되고,
과제명/주제명은 보조 텍스트로 작게,
상태 배지는 한눈에 구분되게,
버튼은 가장 오른쪽에 명확히 배치해줘.

예시 구조:
- 고양이 얼굴 출력
  1주차 과제
  [미제출]   [바로 풀기]

- 두 수의 합
  1주차 과제
  [진행중]   [이어서 하기]

- 문자열 그대로 출력
  1주차 과제
  [완료]     [다시 보기]

행 간 간격은 너무 넓지 않게 하고,
학생이 빠르게 여러 숙제를 훑어볼 수 있도록 밀도 있게 정리해줘.

--------------------------------------------------
6. 오늘의 숙제 카드와 상단 배너 연결
--------------------------------------------------

상단 배너에서 '오늘 문제 2개 더 풀면 목표 달성!' 같은 메시지가 보이고 있으므로,
이 메시지와 아래 '오늘의 숙제' 카드가 자연스럽게 연결되도록 해줘.

예를 들어:
- 상단 배너의 CTA 버튼 '다음 문제 풀기'를 클릭하면
  오늘의 숙제 리스트 중 가장 우선순위가 높은 미제출 문제로 이동하도록 설계
- '이어서 학습하기' 버튼은 진행중 상태의 숙제로 연결

즉, 배너와 숙제 카드가 서로 다른 요소가 아니라 하나의 학습 흐름처럼 보이게 만들어줘.

--------------------------------------------------
7. 숙제가 없을 때 빈 상태 처리
--------------------------------------------------

오늘의 숙제가 없는 경우에는 단순히 빈 리스트로 두지 말고, 아래처럼 빈 상태 UI를 구성해줘.
- 체크 아이콘 또는 완료 아이콘
- 문구: 오늘의 숙제를 모두 완료했어요
- 보조 문구: 이제 추천 문제를 풀거나 지난 문제를 복습해보세요
- 버튼:
  - 추천 문제 풀기
  - 복습하러 가기

즉, 숙제가 없을 때도 대시보드가 죽은 화면처럼 보이지 않게 해줘.

--------------------------------------------------
8. 오늘 학습 현황과 오늘의 숙제의 역할 분리
--------------------------------------------------

반드시 아래 역할을 분리해줘.

오늘 학습 현황:
- 오늘 학습이 얼마나 진행되었는지 보여주는 요약 카드

오늘의 숙제:
- 지금 눌러서 실제로 풀 문제를 보여주는 실행 카드

두 영역을 합치지 말고, 서로 다른 목적을 가진 카드로 유지해줘.

--------------------------------------------------
9. 정보 위계 및 시각적 우선순위
--------------------------------------------------

학생 입장에서 가장 중요한 흐름은 다음 순서가 되어야 한다.
1. 오늘 목표 확인
2. 남은 숙제 수 확인
3. 지금 풀 문제 선택
4. 바로 학습 시작

따라서 시각적 우선순위도 이 흐름에 맞춰줘.
- 상단 배너: 동기부여 + CTA
- 오늘 학습 현황: 진행률 요약
- 오늘의 숙제: 실제 실행 목록

오늘의 숙제 카드는 학생이 가장 자주 누르게 되는 영역이므로,
버튼과 상태 표현이 명확하고 사용성이 좋아야 한다.

--------------------------------------------------
10. 최종 목표
--------------------------------------------------

최종적으로 학생 대시보드는 아래처럼 동작해야 한다.

- 학생이 들어오면 상단 배너에서 오늘 목표를 확인한다
- 오른쪽 상단의 오늘 학습 현황에서 오늘 몇 개를 했는지 본다
- 바로 아래 오늘의 숙제 카드에서 실제로 풀 문제 목록을 본다
- 가장 위에 있는 미제출 문제의 '바로 풀기'를 눌러 즉시 학습을 시작한다

즉, 오늘의 숙제는 반드시 별도 카드로 분리하고,
현재의 요약 카드와 실행 리스트를 기능적으로 분리해서
학생이 더 빠르게 이해하고 행동할 수 있는 대시보드로 개선해줘.
